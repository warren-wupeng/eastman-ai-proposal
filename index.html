03 / Pain Points
核心痛点
品牌素材散落各处，内容创作效率低，多渠道分发困难
📋

素材分散无序

品牌文档、图片、视频散落在个人电脑和各类文件夹中，缺乏集中管理机制，找素材全靠记忆和微信传文件

⏲

内容创作低效

门店依赖总部提供脚本和 demo 视频，自主创作专业度不足，难以针对不同渠道快速产出差异化内容

🔓

合规风险

门店自创内容无法全覆盖审核，存在违反广告法表述风险，品牌调性难以统一保障

06 / Module 1
智能素材库
AI 驱动的品牌数字资产管理平台（DAM）
功能说明
多品牌资产库 按品牌独立分区，管理现有资产类型：品牌宣传视频、访谈视频、活动视频、文案脚本、官方图文帖、活动海报等
AI 智能打标 基于品牌知识库定制训练，上传即识别：品牌归属、产品系列（前挡/侧后/天窗）、适用车型、内容类型、适用渠道
智能搜索 自然语言搜索，如 "龙膜畅悦70 适合小红书的对比图"，支持相似图搜索
版本管理 同一素材多版本追踪，标记最新/归档/废弃，防止使用过期素材
双角色权限 内部角色（管理员/品牌经理/编辑/查看者）+ 外部角色（门店管理员/店员），数据按权限隔离，支持 800+ 账号
使用统计 下载量、使用频次、热门素材排行，数据驱动内容策略

07 / Module 2
AI 内容工厂
AI 能力按客户期望分三层交付：文案（必须）、图片（尽量）、视频（基础）
优先级功能说明
P0 必须 渠道适配文案 选素材 + 选渠道（抖音/小红书/微信），AI 生成种草文、测评文、朋友圈话术，支持多风格
P0 必须 批量变体生成 一键生成多个文案变体（不同角度/风格），方便 A/B 测试和多账号分发
P1 尽量 AI 图片排版 基于品牌模板智能排版小红书笔记、公众号图文、活动海报，适配多渠道尺寸
P2 基础 视频快剪 门店社媒场景：从宣传长视频截取片段、叠加字幕/水印、调整竖版比例。工具辅助，非 AI 生成
AI 辅助合规预检 规则引擎 + AI 双重检测广告法违规风险，标注并建议修改，辅助人工审核

08 / Architecture
技术架构
全球最优模型组合，云原生架构，弹性扩展
用户层
💻 PC Web 管理后台（内部团队） 🌐 门店 Portal（800+ 外部账号） 📱 移动端（远期）
中间件
🛡 API 网关 🌐 CDN 加速分发 ⚡ 异步任务队列 🔒 多租户数据隔离
应用层
🗃 素材管理服务 ⚡ AI 内容生成 🚀 分发管理服务 📖 品牌知识库 🔓 合规引擎
AI 层
🧠 Claude Sonnet (视觉+生成) 🔍 OpenAI Embedding (语义检索) ⚡ Claude Haiku (合规预检) 🔀 LiteLLM 多模型网关 (Phase 2)
数据层
📦 对象存储 (OSS) 📈 向量数据库 🗃 关系型 DB

技术选型要点

  • AI 主力：Claude Sonnet (打标+文案+脚本) + OpenAI Embedding (语义检索)，MVP 仅需集成 2 个 API
  • 多模型路由：Phase 2 引入 LiteLLM 网关，支持 fallback 和成本优化（DeepSeek 低成本备选）
  • 异步 + 流式：任务队列处理 AI 耗时操作，SSE 流式输出缓解文案生成等待体感
  • 存储 + CDN：云存储 + CDN 加速分发，覆盖全国 4000+ 门店
